## 二、基于现有代码的改造清单（字段/组件级）

### A. `web/index.html` 结构改造

#### 1) 页面骨架重排
- 保留顶层 `wrap`，新增语义分区：
  - `app-header`（全局栏）
  - `workspace-main`
  - `workspace-side`
- 现有 `hero` 与 `chat-layout` 合并为统一工作台，减少重复层级。

#### 2) 现有块位移建议
- `healthSummary`（运行摘要）移动到右侧栏顶部常驻卡片。
- `helpPanel` 从“快捷操作卡片内部”改为可折叠面板（统一组件）。
- `securityPanel` 从聊天框下方浮层式区域改为“对话流内系统卡片”渲染（更连续）。

#### 3) 标签页策略
- 当前 `tabChat/tabNotes/tabConfig/tabSystem/tabAdmin` 改成：
  - 聊天页常驻
  - 其他页进入右侧抽屉/全屏面板（降低跳页割裂感）

### B. `web/index.html` 样式体系重构

#### 1) 增加设计令牌（CSS 变量）
- 新增 token 分组：
  - `--color-*`（语义色）
  - `--space-*`（间距）
  - `--radius-*`（圆角）
  - `--shadow-*`（阴影）
  - `--font-size-*`（字号）

#### 2) 消息样式统一
- 现有 `.msg.user/.assistant/.system/.error` 改为：
  - 统一基类 `message-card`
  - 语义修饰 `is-user/is-assistant/is-system/is-error`
- 增加消息头（角色 + 时间）和正文分区，避免“大段纯文本块”。

#### 3) 按钮系统统一
- 现有 `button` + `.secondary` 重构为：
  - `btn-primary / btn-secondary / btn-ghost / btn-danger`
- 禁用态、悬浮态、加载态统一视觉规则。

### C. `web/app.js` 交互逻辑重构

#### 1) 聊天状态机标准化
- 当前 `chatState` 扩展为固定枚举：
  - `idle` / `sending` / `awaiting_access_code` / `error`
- 所有入口统一调用 `setChatState`，并由状态驱动 UI（按钮禁用、文案、面板显隐）。

#### 2) 受限查询流程组件化
- 保留现有函数能力：
  - `submitSecurityKey()`
  - `abortAccessTicket()`
  - `closeSecurityPanelAction()`
- 但渲染方式改为“消息流中的验证卡片”，避免上下文断裂。

#### 3) 命令结果模板化
- `handleLocalCommand()` 的结果渲染统一为 `appendCommandResultCard(type, title, body, meta)`。
- `/config`、`/metrics`、`/files` 输出改“结构化块”而非单段文本。

#### 4) 重建索引反馈降噪
- `triggerRebuildAndWait()` 中“循环 appendMessage”改为：
  - 聊天区只记录关键状态（开始/成功/失败）
  - 详细进度写入侧栏 `RebuildProgressCard`

### D. 可访问性与文案规范（两文件联动）

- 键盘支持：
  - `Enter` 发送、`Shift+Enter` 换行（输入框可升级为 textarea）
  - `Esc` 关闭帮助/抽屉
- `aria-label` 完整覆盖关键控件（发送、关闭、重建、提交口令）。
- 系统提示文案统一成“动作 + 结果 + 下一步”。

---

## 三、建议落地顺序（低风险）

1. **阶段 1（结构+视觉）**
   只改 `web/index.html` + CSS，不动 API 协议。
2. **阶段 2（状态机+渲染）**
   重点改 `web/app.js` 的消息渲染与状态驱动。
3. **阶段 3（打磨）**
   移动端、细节动效、错误文案统一、可访问性补齐。
